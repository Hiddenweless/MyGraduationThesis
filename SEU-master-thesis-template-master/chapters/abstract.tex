\begin{abstract}{图推理, 大语言模型, 多智能体}
图推理是图数据分析与智能决策中的关键问题，在路径规划、网络分析、组合优化和知识推断等场景中具有重要应用价值。近年来，大语言模型在复杂推理和程序生成方面取得显著进展，为图推理任务的自动化求解提供了新的可能。然而，图推理具有拓扑依赖强、约束条件多和答案可验证性要求高等特点。现有端到端方法通常将图数据序列化为文本后直接推理，容易在长序列和复杂约束下出现拓扑关系遗漏、结构理解偏差与答案幻觉；依赖历史样本、经验代码库或专门微调的程序生成方法虽然能提升局部表现，但在构建成本、知识更新和跨任务泛化方面仍存在局限。

针对上述问题，本文提出基于规划反馈的图推理多智能体协同求解框架 PlanGraph，将求解过程组织为规划生成、规划引导检索和执行验证反馈三个阶段。问题智能体与规划智能体解析自然语言任务并生成结构化伪代码规划，显式表达任务目标、核心算法步骤和约束条件；检索智能体依据规划结果从图算法文档知识库中检索相关 API 与实现知识；编码智能体基于规划和检索结果生成可执行程序，并结合执行测试、错误反馈、推理修正和有界重生成形成验证闭环。规划约束知识调用与程序生成，反馈修正实现偏差和约束遗漏。

本文在 GraphArena、NLGraph、GraphWiz、LLM4DyG 和 GraphInstruct 五个公开图推理基准测试集上进行实验。PlanGraph 在其中四个基准上取得最优结果，平均精确匹配率达到 96.89\%，总体优于现有方法，并在经典图算法、组合优化和指令式图推理任务中表现出较强鲁棒性。实验结果表明，将显式规划、外部知识检索与程序验证组织为反馈闭环，能够缓解大语言模型在图推理任务中的幻觉与不稳定问题，为大语言模型和图算法工具协同求解复杂结构化任务提供可运行、可解释的实现路径。
\end{abstract}

\begin{englishabstract}{graph reasoning, large language models, multi-agent systems}
Graph reasoning is a key problem in graph data analysis and intelligent decision-making, with applications in path planning, network analysis, combinatorial optimization, and knowledge inference. Large language models (LLMs) have shown strong abilities in reasoning and program generation, but graph reasoning still requires reliable use of topological relations and task constraints. Existing end-to-end methods usually serialize graph data into text, which may cause missing relations, biased structural understanding, and hallucinated answers under long inputs. Program-generation methods based on historical samples, code repositories, or task-specific fine-tuning can improve local performance, but they still face high construction costs and limited cross-task generalization.

To address these problems, PlanGraph, a planning-feedback-based multi-agent collaborative framework for graph reasoning, is proposed. PlanGraph organizes the solving process into planning generation, plan-guided retrieval, and execution-verified feedback. The question agent and planning agent transform natural language graph tasks into structured pseudocode plans describing objectives, algorithmic steps, and constraints. The retrieval agent retrieves relevant APIs and implementation knowledge from a graph algorithm documentation knowledge base according to the plan. The coding agent generates executable programs and forms a verification loop through execution testing, error feedback, reasoning revision, and bounded regeneration. Planning constrains knowledge invocation and program generation, while feedback corrects implementation deviations and omitted constraints.

Experiments are conducted on five public benchmarks. PlanGraph achieves the best performance on four of them, with an average exact match rate of 96.89\%. Overall, it outperforms existing methods and shows strong robustness in classical graph algorithms, combinatorial optimization, and instruction-style graph reasoning. The results demonstrate that organizing explicit planning, external knowledge retrieval, and program verification into a feedback loop can alleviate hallucination and instability in LLM-based graph reasoning, providing an executable and interpretable path for solving complex structured tasks with LLMs and graph algorithm tools.
\end{englishabstract}
