\chapter{智能体提示词与检索查询模板}
\label{appendix:prompt_templates}

为保证 PlanGraph 系统的可复现性，同时避免堆叠过长的工程提示词文本，本文仅给出各智能体提示词和检索查询模板的摘要形式。需要说明的是，问题智能体、规划智能体、编码智能体和推理智能体采用提示词驱动；检索智能体不调用大语言模型，而是根据规划工件、任务约束和执行反馈构造检索查询。

\section{检索查询模板摘要}

检索智能体的作用是根据规划结果和反馈信息组织检索查询，而不是生成自然语言回答。其查询构造主要利用任务类型、图属性、算法提示、约束标签、输出格式以及执行错误信息。该设计使检索过程不直接依赖原始问题文本，而是围绕当前求解步骤和约束条件展开。表\ref{tab:appendix_retrieval_query_template}给出了检索查询模板的摘要。

\begin{table}[H]
\centering
\small
\caption{检索查询模板摘要}
\label{tab:appendix_retrieval_query_template}
\begin{tabular}{p{2.0cm}p{10.5cm}}
\toprule
查询类型 & 构造依据 \\
\midrule

\section{核心提示词模板摘要}

本节仅保留各提示词中影响系统行为的关键约束。实际实现中，各模板还包含输出格式检查、异常处理说明和少量工程性规则。


问题智能体用于将原始自然语言问题解析为结构化图推理任务，输出字段包括 \texttt{task\_type}、\texttt{nodes}、\texttt{edges}、\texttt{directed}、\texttt{weighted}、\texttt{capacity}、\texttt{params}、\texttt{output\_format} 和 \texttt{constraints}。其核心模板如下：

\begin{quote}
\small
\ttfamily
Extract a graph reasoning task as strict JSON.\\
Fields: task\_type, nodes, edges, directed, weighted, capacity,\\
params, output\_format, constraints.\\
Use edges as arrays [u, v] or [u, v, weight\_or\_capacity].\\
Question: \{question\}
\end{quote}

规划智能体用于生成算法级伪代码规划，并为检索阶段提供算法提示、约束标签和检索术语。其核心模板如下：

\begin{quote}
\small
\ttfamily
Produce structured algorithm-level pseudocode, not Python code.\\
Include graph construction, algorithm choice, constraint checks,\\
and output formatting.\\
Do not copy concrete node names, edge lists, or weights into the plan.\\
Return JSON with: pseudocode, constraints, algorithm\_hints,\\
retrieval\_terms, constraint\_tags, expected\_api\_behavior.
\end{quote}

编码智能体用于基于任务、规划和检索知识生成可执行 Python 程序。其核心模板如下：

\begin{quote}
\small
\ttfamily
Generate a complete Python program body.\\
Runtime variable graph\_data already exists.\\
Use standard library and NetworkX only.\\
Build all nodes and edges from graph\_data.\\
Do not hardcode node names, edge lists, or weights.\\
Use retrieved knowledge only when it matches the graph type,\\
constraints, API inputs, return values, and output format.\\
The script must execute at top level and print only the final answer.
\end{quote}

推理智能体用于根据执行反馈和检索反馈修正当前程序，主要处理图构建错误、API 使用错误、约束遗漏和输出格式不匹配等问题。其核心模板如下：

\begin{quote}
\small
\ttfamily
Fix the code using execution feedback and retrieval feedback.\\
Keep graph\_data as the input variable.\\
Revise API misuse, missing constraint handling,\\
incorrect graph construction, or output mismatch.\\
Use the plan, retrieved knowledge, and execution feedback for repair.\\
The fixed script must execute at top level and print only the answer.
\end{quote}
